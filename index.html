Project Steps

The World Layoffs Data Cleaning Project Steps

I first inserted my excel layoffs.csv and created a different table called "layoffs_staging" for the purpose of not corrupting the original raw data incase we still need it. I did this by inserting all data informations from my original laysoff to my layoffs_staging table. I took 4 steps to clean to my data, this include:
Step 1: Remove all the duplicates to reduce error and increase reliability and accuracy of the data.

Step 2: Standardizing the data which means I will check for the format and make sure it is consistent with the rest of the data. I removed or fixed any formatting that aren't uniform to encourage consistency and quality that is important for comparison when analyzing the data.

Step 3: Populate or remove any nulls or blanks value that are unneccessary and can cause bad data quality. This step is needed to increase data integrity.

Step 4: Remove all unneccessary columns to reduce redundancy and efficiency for data anlysis.

Step 1: Remove Duplicates

In this step, I checked for any duplicates by selecting the layoffs_staging table and looking at all the rows of each columns. I decided to create another table called "layoffs_staging2" from the "layoffs_staging" table to delete the duplicates and avoid redundancy processes.

I used ROW_NUMBER() and made a new column named "row_num" as well as paritioning each column in my data to list the numbers of duplicates. For example, any row_num listed that is greater than (>) 1 gets removed. I checked to see if my code works by selecting "layoffs_staging" and a random company "Casper." It worked! There's only one result for Casper so no duplicates!

Step 2: Standardizing the Data

In Step 2, I standardize the data that I previously saw were not uniformed and consistent. This part is important for data integrity and accuracy! I fixed the white space before each company name through TRIM(). I removed the typos such as the "%" after the "Crypto" industry and removed the "." at the end of "United States" country. Now this makes the data so much more reliable to use!

I made sure to change the formatting of the date so it's uniformed to (mm/dd/yyyy).

Step 3: Null and Blank Values

For Step 3, I checked the columns where I noticed there were Null and Blank values. In the percentage_laid_off and total_laid_off, I didn't remove or populate the null values yet because null is an absence of value, the company didn't imply the # so it's unknown, I cannot populate a random number or remove it for the consistency of the data without introducing misinformation and alter results for decision-making. However, I did populate the Null and Blanks Value in the industry column. I did this by using the same existing Company's industry data and populate the same company that has their industry as Null or Blank. I made all Blank values into Null values before I populate all Nulls.

I populated the NULL with the correct matching industry, where company 1 and 2 matches each other. I also used the company "Airbnb" to check if my populating worked, and it did! Airbnb now shows that the industry is "Travel" (one of the Airbnb had NULL in their industry row in the beginning).

Step 4: Remove Unneccessary Columns

In Step 4, I removed the percentage_laid_off and total_laid_off columns because of the NULL values. I also remove the row_num column because it was no longer needed for the final result.
